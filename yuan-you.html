随着 Web 应用的发展，前端 Single Page Applications（SPA, 单页应用程序）越来越流行，“前后端分离”开发模式也越来越成熟。以下通过几张图加强对相关概念的理解。

做为“前后端分离”开发模式下的前后端数据通信的“桥梁” —— API，接口的规范化也显得越来越重要，实际项目实践中已遇到以下诸多问题：

  1. 无接口文档，口述沟通
  2. 接口功能不完善就提前进入联调
  3. 接口功能拆分的维度不合理
  4. 接口功能测试依赖前端界面
  5. 接口格式规范存在多种：WCF Server（.svc） 和 Web API
  6. 接口请求方式无语义，比如全部是 POST
  7. 不同状态下，同一接口返回的数据结构不一样
  8. 接口返回的数据值类型不统一，有些为 String，有些为 Number
  9. 接口状态标识及业务返回码混乱
  10. 接口数据变更没有通知相关前端人员
  11. 各项目组接口定义规则因人而异（比如：接口的命名规则；字段的命名规则；标识的规则；分页、统计、超时、授权通用功能设计...）
  12. ...

基于上述原因，所以“前后端分离”开发模式下的《后端交互规范》文档就显得至关重要了。
